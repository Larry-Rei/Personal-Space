项目描述：基于 GameFramework (GF) 框架开发的通用 UI 活动套件，涵盖签到、抽奖、战令及背包等核心业务模块。

GameFrameworkMVC 架构C#事件系统 (Event)引用池 (Reference Pool)单例模式

⚙️ 核心架构与逻辑拆解

  • UI 架构搭建与 MVC 实践：深度应用 GF UI 组件，基于 MVC 模式实现逻辑、数据与显示的严格分离。通过自定义 UIForm 窗体生命周期管理，规范了 UI 逻辑层的打开、轮询及关闭流程，极大地提升了系统可维护性。
  • 复杂业务逻辑实现：独立设计并开发了 7 日签到、转盘抽奖及 BattlePass 战令系统。采用配置表驱动设计（Data-Driven），通过统一的逻辑控制器处理多重奖励判定与状态同步，确保了业务逻辑的健壮性。
  • 高性能背包系统与引用池：针对背包中高频生成的物品格，运用 GF 引用池 (Reference Pool) 技术进行对象管理，有效规避了频繁 Instantiate 带来的内存抖动，并配合事件中心实现背包数据与 UI 的异步解耦刷新。
  • 架构优化与解耦设计：灵活运用单例模式维护全局数据中心，并利用 GF 事件系统实现各个活动模块间的“零耦合”通信。通过接口隔离与引用回收机制，显著降低了各功能模块间的依赖复杂度。
